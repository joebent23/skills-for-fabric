### Questions and Discussions

For questions, ideas, or general discussions:

1. **GitHub Discussions** (preferred for community questions)
   - Browse existing discussions
   - Start a new discussion if your question isn't answered
   
2. **GitHub Issues** (for bugs and feature requests)
   - Search existing issues first
   - Use issue templates when creating new issues
   - Provide detailed reproduction steps

### Issue Types

| Issue Type | Where to Report |
|------------|----------------|
| Bug in a skill | GitHub Issues |
| Feature request | GitHub Discussions → Issues if approved |
| Usage question | GitHub Discussions |
| Security vulnerability | **DO NOT** use Issues - see [SECURITY.md](SECURITY.md) |
| Documentation improvement | GitHub Issues |

### Response Times

This is an open-source project maintained by Microsoft employees:

- **Critical bugs**: 1-3 business days
- **Security issues**: <24 hours (see SECURITY.md)
- **Feature requests**: Best effort, no SLA
- **Questions**: Community-driven, no SLA
